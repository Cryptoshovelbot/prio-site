## TL;DR

Trois axes prioritaires :

1. **Forcer redirection `www.getprio.io` -> `getprio.io`** (cause #1 du signal *« Autre page avec balise canonique correcte »*). Aujourd'hui www sert le meme HTML que l'apex avec un canonical pointant ailleurs : Google exclut chaque page www correctement, mais ces exclusions polluent le rapport de couverture.
2. **Tuer pour de bon les deux cannibalisations actives** -- `barcelona-rental-market-2026-prices-trends` (212 impressions, 0 click, redirige vers le winner) et `documents-rent-apartment-spain-checklist` (vs le winner `documents-rent-apartment-spain` qui fait 581 impressions, 0 click). Cause #1 du signal *« Page avec redirection »* + autorite divisee.
3. **Recuperer le CTR effondre.** Position moyenne grimpe de 14.5 -> 7.3 (excellent), mais CTR chute de 0.81% -> 0.38%. Les pages remontent dans les SERP sans capturer le clic -> retravailler titles + metas des 5 top pages par impressions.

Croissance globale : +238% impressions (619 -> 2093) sur fond de remontee massive de la position moyenne.

## 2. Diagnostic des 2 signaux GSC remontes

### *« Page avec redirection »*

Sources confirmees par test live :

| URL | Code | Cible |
|---|:-:|---|
| `/blog/barcelona-rental-market-2026-prices-trends/` | **301** | `/blog/barcelona-rental-market-2026/` |
| `/home` | 301 | `/` |
| `/barcelona` | 301 | `/es/barcelona/` |
| `/madrid` | 301 | `/es/madrid/` |
| `/valencia` | 301 | `/es/valencia/` |
| `/spain` | 301 | `/es/` |
| `/blog/nie-number-spain-guide` (sans slash) | 308 | `/blog/nie-number-spain-guide/` |
| `/es/barcelona` (sans slash) | 308 | `/es/barcelona/` |
| `http://getprio.io/` | 301 | `https://getprio.io/` |
| `/www.getprio.io/*` (regle dans `_redirects`) | -- | **regle cassee : Cloudflare ne resout pas les hostnames dans les paths** |

**Diagnostic principal :** L'URL cannibalisee `barcelona-rental-market-2026-prices-trends` continue de generer 212 impressions a position 6.9 cinq semaines apres avoir ete redirigee. Google indexe encore l'URL meme avec 301 active. Cause probable : le fichier `index.html` existe toujours dans le repo (`/opt/prio-site/blog/barcelona-rental-market-2026-prices-trends/index.html`) et meme si Cloudflare `_redirects` doit prevaloir, Google a un cache historique a purger.

**Sources secondaires :** les 5 alias courts (`/home`, `/barcelona`, `/madrid`, `/valencia`, `/spain`) fonctionnent, ne sont pas dans le sitemap, n'ont aucun lien interne (`grep` confirme). Mais s'ils etaient linkes depuis l'exterieur ou des anciens caches, ils apparaissent comme « Page avec redirection ».

**Trailing slash 308 :** Cloudflare Pages ajoute systematiquement le trailing slash. AUCUN lien interne ne pointe sans slash final (verifie sur tout le site -- 0 occurrence). Donc ces 308 ne viennent pas de notre code, mais d'eventuels liens externes / canonicalisations historiques.

### *« Autre page avec balise canonique correcte »*

**Cause #1 confirmee : duplicat www / apex.**

```
$ curl -sI https://www.getprio.io/
HTTP/2 200          # pas de redirect

$ curl -s https://getprio.io/  | md5sum
b2e70b645924866dd8bfa72fbbb72b43

$ curl -s https://www.getprio.io/  | md5sum
b2e70b645924866dd8bfa72fbbb72b43   # contenu identique
```

Le HTML de www contient `<link rel="canonical" href="https://getprio.io/...">` (sans www). Google :
1. Decouvre la version www (probablement DNS / mention externe)
2. Constate le contenu identique
3. Voit le canonical qui pointe ailleurs
4. Range correctement sous *« Autre page avec balise canonique correcte »*

C'est techniquement « OK » mais ca pollue le rapport de couverture. Confirme dans les donnees : `https://www.getprio.io/es/valencia/` apparait dans le top pages GSC (1 impression, position 11). Si Valencia y est, **toutes les pages du site** doivent y etre theoriquement.

**Cause #2 possible :** la page cannibalisee `barcelona-rental-market-2026-prices-trends` -- si le redirect Cloudflare echoue dans certains cas, le fichier statique se serve, et son canonical pointe... vers lui-meme (`https://getprio.io/blog/barcelona-rental-market-2026-prices-trends/`). Donc PAS un cas de canonical-vers-autre-URL. Donc PAS la source du signal #1. La supprimer du repo eliminera quand meme l'ambiguite.

## 4. PLAN D'ACTION PRIORISE

### ACTION 1 -- Forcer redirection www -> apex (CRITIQUE / Effort FAIBLE)

**Pourquoi :** elimine la totalite des entrees « Autre page avec balise canonique correcte » dues au duplicat www, soit potentiellement 40 pages.

**Comment :** `_redirects` Cloudflare Pages ne supporte PAS les patterns hostname (la regle `/www.getprio.io/*` actuellement en place ne sert a rien). Trois options :

1. **Recommandee :** Cloudflare Dashboard -> domaine `getprio.io` -> Rules -> Redirect Rules. Creer une regle qui match `Hostname equals www.getprio.io` -> redirect 301 vers `https://getprio.io${URI}`.
2. Bulk Redirects au niveau zone Cloudflare.
3. Si Cloudflare Pages > Custom Domains : verifier que `www.getprio.io` est configure comme redirect au niveau Pages (parametre disponible dans les versions recentes).

**Verification post-deploiement :**
```bash
curl -sI https://www.getprio.io/      # doit retourner 301 -> https://getprio.io/
curl -sI https://www.getprio.io/es/   # doit retourner 301 -> https://getprio.io/es/
```

**Cleanup associe :** retirer la regle inutile `/www.getprio.io/* https://getprio.io/:splat 301` du fichier `_redirects` (ligne 8).

### ACTION 2 -- Finaliser la consolidation `barcelona-rental-market-2026` (CRITIQUE / Effort FAIBLE)

**Pourquoi :** 212 impressions/mois bloquees sur une URL redirigee. Combinees au winner (293 impressions), c'est ~500 impressions/mois pour la meme thematique. Position 6.9-7.9 -> on peut viser pos 5 avec autorite consolidee.

**Comment :**
1. Verifier que le contenu du winner (`barcelona-rental-market-2026/index.html`) absorbe tous les angles uniques de l'ancien (donnees prix, tendances). Si oui, etape 2 ; si non, fusionner d'abord.
2. **Supprimer** `/opt/prio-site/blog/barcelona-rental-market-2026-prices-trends/index.html` du repo (le redirect Cloudflare reste actif via `_redirects`).
3. GSC -> Index -> Suppressions -> Demande temporaire pour cette URL (acceleration de la desindexation).
4. Garder la 301 dans `_redirects` au moins 6 mois.

### ACTION 3 -- Consolider `documents-rent-apartment-spain` vs `-checklist` (CRITIQUE / Effort MOYEN)

**Pourquoi :** la page top du site en impressions (581 impr en 35j, 0 click, pos 7.1). Cannibalisee depuis le debut, jamais traitee. Title et meta tres proches (« Renting in Spain: Documents You Need » vs « Documents to Rent in Spain: 2026 Checklist »).

**Comment (meme procedure que Action 2) :**
1. Choisir le winner. Recommandation : garder `documents-rent-apartment-spain` (URL plus courte, title plus naturel, top GSC).
2. Fusionner le meilleur du contenu de `-checklist` dans le winner (tableau checklist explicite).
3. Ajouter au `_redirects` :
   ```
   /blog/documents-rent-apartment-spain-checklist/  /blog/documents-rent-apartment-spain/  301
   /blog/documents-rent-apartment-spain-checklist   /blog/documents-rent-apartment-spain/  301
   ```
4. Supprimer le fichier `documents-rent-apartment-spain-checklist/index.html`.
5. Retirer du sitemap `/opt/prio-site/sitemap.xml` (ligne 30).
6. GSC -> Suppressions.

### ACTION 4 -- Recuperer le CTR (HAUT / Effort MOYEN)

**Comment :** un script qui regenere le sitemap a partir des `mtime` reels des `index.html`. Idealement automatise au build/deploy.

### ACTION 7 -- Reduire la cardinalite « Page avec redirection » (BAS / Effort FAIBLE)

Liste des redirections justifiees a garder : `/home`, alias villes -> ES, `/blog/barcelona-rental-market-2026-prices-trends/`, futurs alias `/documents-rent-apartment-spain-checklist/`. Ces URLs continueront d'apparaitre dans GSC tant qu'elles sont referencees ailleurs ; c'est normal pour des redirections legitimes.
